# skill 搜索为主的知识库

我要以 skill 搜索为主做一个知识库搜索，知识库包含录入数据(上传文档、ai 归类、 使用 skill-creater 写入数据库)，搜索，使用系统 skill来进行搜索(执行 sql 语句的方式，类似于文件检索能力（ls、grep、glob），这里我需要把 skill 存入数据库中，方便后续搜索，毕竟文件检索的效率太低了)

1. 能够看到上传文档的列表，以及文档的分类情况
2. 能够看到所有的 skill 列表，以及 skill 的分类情况，以及 skill 的内容和文档内容
3. 需要实现 系统 skill 能力，通过 sql 语句来搜索 skill 和文档内容，以适合渐进式披露的目的

我需要参考 embedease-ai 项目的实现
1. 参考 embedease-ai 项目中客户聊天部分的实现，也要统一使用 markdown 作为唯一交互格式，使用富文本编辑器
2. 参考 embedease-ai 项目中langchain 的实现，也要使用统一的交互协议，使用 langchain v1 的返回模式，前后端都要
3. 参考 embedease-ai 项目中数据库的实现，也要使用统一的数据库，使用 sqlite3
4. 本项目仅仅只有系统级别的 skill 和 文档 skill，文档 skill 支持查看和修改，但是系统级别的不能。 文档 skill 能够支持迁移到不同文件夹(文件夹归类的关系在数据库中记录)
5. 参考 embedease-ai 项目中 提示词管理，所有系统提示词要统一放到数据库中，后续能够查看和编辑，也要统一使用 markdown 作为唯一交互格式，使用富文本编辑器
6. 参考 embedease-ai 项目中一进入项目中的快速设置页面，我也需要把必须填入的内容，比如 ai key，使用 dev.model 才能够跑起来(用户最小填入规则)，要有测试，测试通过才能下一步。
